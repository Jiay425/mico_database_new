# 业务工作流定义

## 患者分析工作流

标准顺序：

1. 获取患者详情
2. 获取当前样本
3. 获取样本 Top 特征
4. 获取健康组参考
5. 获取预测结果
6. 汇总解释

解释原则：

- 先讲当前样本
- 再讲与健康组的偏离
- 再讲预测是否与偏离同向
- 最后才给出建议

## 首页分析工作流

标准顺序：

1. 获取 dashboard 摘要
2. 识别主要疾病与群体结构
3. 提取地区与年龄性别重点
4. 生成简洁结论

## 预测解释工作流

标准顺序：

1. 获取预测载荷与预测结果
2. 读取 Top1 / Top2 概率
3. 结合健康组偏离解释
4. 明确模型边界
5. 给出下一步建议

## 关键约束

- 所有预测必须以单一样本为单位
- 所有健康对照优先使用相同 `body_site`
- 当同部位健康样本不足时，退化到全体健康组
- 任何结果都不能直接表述为最终医学诊断
